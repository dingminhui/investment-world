腾讯Hy3：小参数大能力——底层技术拆解与对持仓的含义
背景：市场评价称，Hy3 正式版相比2个月前的 preview 参数规模完全未变——总参数295B、激活仅21B，是国内外一线模型中最小的；但它在12项重点基准中7项进入主要对比模型前三，几乎在所有主要指标超过 DeepSeek V4 Pro，大部分指标超过 Seed2.1 Pro 与 Qwen3.7。本文拆解：腾讯如何用这么小的参数做到如此强的能力。

一句话答案

低成本靠 MoE（混合专家），强能力靠「后训练(RL) + 架构细节 + 真实场景数据闭环」三件套。参数规模两个多月没变却明显变强，恰恰证明真正拉开差距的不是参数，而是后面那些。

一、295B / 21B 这个数字到底什么意思（低成本的根源）

Hy3 是 MoE（混合专家）架构，规格来自腾讯混元团队官方文档（2026-07-06 发布，preview 为4月底，参数未改）：

维度数值含义
总参数295B整个模型的"知识储备"
激活参数21B（仅~7%）每次推理实际参与计算的参数
专家数192个，Top-8激活每道题只唤醒最相关的8个专家
层数80层—
注意力GQA（64 Q头 / 仅8 KV头）KV缓存极小 → 长上下文便宜
上下文256K长文档 / 多轮 Agent
MTP层3.8B（多token预测）推测解码 → 解码更快
精度BF16，附FP8量化版8张H20即可部署

MoE 的核心思想：不是每个问题都需要全部知识。模型像一个192人的专家库，路由网络每次只叫醒最对口的8个人。所以——

你付的是 21B 的电费，享受的是 295B 的服务。

这就是"参数只有别人的1/3到1/5，推理成本却最低"的真相：295B 是存储，21B 才是算力开销。稠密（dense）模型每 token 要把全部参数跑一遍，而 Hy3 只跑 7%。

二、但 MoE 解释不了"为什么能上桌吃饭"

如果只靠 MoE，Hy3 应该只是"便宜的21B水平"。它能压着 DeepSeek V4 Pro、追平 Seed2.1 / Qwen3.7，靠的是另外三件事——而且官方文档把这点说得很直白：相比2个月前 preview，参数一字未改，改的全是后训练。

1. 后训练 RL 放大 + 数据质量升级（最关键的一棒）

官方原话："improved the quality and diversity of post-training data while scaling up RL training"。2024年以来，模型在真实任务上的能力跃升，主要不来自堆参数，来自强化学习（推理训练）。一个21B激活、但后训练充分的模型，可以打赢没做推理训练的大模型。

2. 快慢思考融合

reasoning_effort 参数：简单问题默认 no_think 直接答（低延迟低费），数学/代码/复杂推理才拉到 high 启动深度思考链。只在需要时才付"思考"的成本——这是把 MoE 的"按需激活"思想从参数层面延伸到推理层面。

3. 生产级可靠性（幻觉/工具调用/多轮）

• 幻觉率 12.5% → 5.4%，常识错误 25.4% → 12.7%（细粒度数据清洗 + "有依据才答、无依据明说、不编造"的训练约束）
• 工具调用稳定性、错误自恢复、跨脚手架泛化（SWE-Bench Verified 不同脚手架方差 <4%）
• 多轮意图跟踪错误率 17.4% → 7.9%

这些东西不上排行榜，但决定一个模型能不能真正进产品。Hy3 明显是冲着"好用"而非"跑分"调的。

三、一个必须说清楚的诚实点：1/5 这个数字有水分

提醒：你引用的对比是 Hy3 的 295B 总参数 vs DeepSeek V4 Pro 1.6T、Qwen3.7 Max 1.2T 的总参数。问题在这里——那些"万亿级"模型，大概率本身也是 MoE。既然是 MoE，它们的"激活参数"远小于总参数（就像 Hy3 自己 295B 总 / 21B 激活）。

所以——

• 总参数 295B vs 1.6T 的"1/5"对比，是拿苹果的总重比橙子的总重；
• 真正决定成本和单 token 能力的，是激活参数。如果对手激活参数也只有几十B，那 Hy3 的"效率优势"远没有5倍那么夸张；
• 腾讯官方自己的措辞是"rivals flagship models with 2-5x the parameters"——这个 2-5 倍是公允的（对稠密旗舰、或总参更大的 MoE 都成立）。

准确说法：Hy3 用"modest 的总参 + 极小的激活 + 强后训练"，在实用任务上追平了激活参数可能并不比它大太多的更大模型。MoE 是前提，强后训练是临门一脚。

四、作为腾讯股东，这件事的真正含义

这正好接上我们之前聊的"腾讯 GPU 投入产出"那条线：

1. 21B 激活 = 推理成本 ≈ 21B 稠密模型 → API 定价可以很低、同时还有毛利 → AI 是利润中心，不是纯成本黑洞。这正是当初判断"腾讯风险比 Meta 小"的底层支撑之一。

2. 真正的护城河不是模型权重，是"生态数据闭环"。Hy3 由姚顺雨（首席AI科学家）主导、201天重建架构；接入元宝/ima/QQ浏览器/微信读书/腾讯游戏等50+产品团队，真实产品遥测反哺后训练。官方一句"用更小的参数量撬动更高的实用价值"，道尽的正是这条飞轮。

3. 给你的投资提醒（也接"安全边际"那篇）：模型能力正在快速 commoditize（Hy3 本身 Apache 2.0 开源免费商用）。值得长期下注的，不是"混元比别人强多少"，而是微信/元宝/ima 的分发霸权 + 场景数据 + Agent 产品化能力。权重会贬值，闭环不会。

五、小结

Hy3 用"小参数 + 强后训练"证明了一件事：在模型能力上，工程与数据闭环的杠杆，已经压过了堆参数的杠杆。对腾讯股东而言，这降低了 AI 军备竞赛的边际成本，同时把胜负手从"谁模型大"转向"谁的生态能把模型用起来"。这和我们之前对腾讯 GPU 投入的判断方向一致——关键不在投入规模，在投入能否通过微信/元宝/ima 的闭环转化为可持续的差异化能力。

资料来源：腾讯混元团队官方文档（cnb.cool / ModelScope / cloud.tencent，发布于2026-07-06）；"2-5x 参数"为官方表述。DeepSeek V4 Pro / Qwen3.7 Max 是否为 MoE 未单独核实，第三部分"万亿模型大概率也是 MoE"是基于行业惯例的推断。
